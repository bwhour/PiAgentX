# 项目重构指南

## 概述

本文档提供了 PI Agent 项目重构的完整指南，包括计划、检查清单、工作流程和工具。

## 文件说明

### 核心文档
1. **PROJECT_REORGANIZATION_PLAN.md** - 完整的重构计划，包含目标、阶段、时间线和资源需求
2. **REFACTOR_CHECKLIST.md** - 详细的检查清单，按阶段和任务组织
3. **REFACTOR_WORKFLOW.md** - 工作流程和 Git 分支策略
4. **REFACTOR_README.md** - 本文件，快速开始指南

### 工具脚本
1. **start-refactor.sh** - 重构启动脚本，自动化初始步骤

## 快速开始

### 1. 检查项目状态
```bash
./start-refactor.sh check
```

### 2. 创建项目备份
```bash
./start-refactor.sh backup
```

### 3. 开始第一阶段重构
```bash
./start-refactor.sh phase1
```

### 4. 查看详细任务
```bash
# 查看第一阶段任务
cat REFACTOR_CHECKLIST.md | grep -A20 "第一阶段：基础重构"
```

## 重构阶段

### 第一阶段：基础重构（1-2周）
- 创建新的目录结构
- 移动 Python 示例到 `examples/`
- 整理文档到 `docs/`
- 重构 `src/` 目录结构

### 第二阶段：功能完善（2-3周）
- 建立测试体系
- 优化配置系统
- 创建构建脚本
- 生成文档

### 第三阶段：高级功能（3-4周）
- 开发插件系统
- 增强 TUI 界面
- 添加性能监控
- 增强安全性

## Git 工作流程

### 分支策略
```
main (稳定版)
  │
  └── develop (开发版)
        │
        ├── feature/refactor-phase1
        ├── feature/refactor-phase2
        └── feature/refactor-phase3
```

### 每日工作流程
1. 从 `develop` 分支创建当日工作分支
2. 实施具体任务
3. 运行测试和验证
4. 合并回阶段分支
5. 更新任务清单

## 验证标准

### 必须通过的检查
- [ ] 所有测试通过
- [ ] TypeScript 编译无错误
- [ ] 基本功能正常
- [ ] 文档链接有效

### 质量指标
- 测试覆盖率 > 80%
- 构建时间减少 20%
- 代码重复率 < 5%
- 无严重安全漏洞

## 风险管理

### 已知风险
1. **代码破坏**：重构可能引入 bug
2. **进度延迟**：复杂任务可能超时
3. **依赖问题**：路径更改导致导入失败

### 缓解措施
1. **小步快跑**：每次只重构一小部分
2. **充分测试**：每个更改都有测试
3. **定期备份**：重要节点创建备份

## 紧急情况处理

### 遇到问题怎么办？
1. **测试失败**：回滚到上一个稳定提交
2. **编译错误**：检查导入路径和类型定义
3. **功能异常**：使用调试工具定位问题

### 回滚流程
```bash
# 回滚到上一个稳定提交
git reset --hard HEAD~1

# 或回滚到特定提交
git reset --hard <commit-hash>
```
